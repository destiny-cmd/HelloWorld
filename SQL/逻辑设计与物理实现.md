逻辑设计基于概念设计，物理设计基于逻辑设计。



## 逻辑设计
### 教学内容
+ 概念设计向逻辑设计的转换规则：
    - 实体→表
    - 1:1联系 → 外键放入任一方
    - 1:N联系 → 外键放入N方
    - M:N联系 → 转换为独立的关系表



### 逻辑设计
+ **核心目标**：将概念模型转化为特定数据模型（通常是关系模型）下的数据结构，使之符合数据库系统的处理要求。
+ **主要产出物**：**关系模式**（即表结构定义）、视图定义、完整性约束规则。
+ **关注视角**：**技术实现的逻辑视角**。重点在于把E-R图中的实体和关系，转换为二维表格（关系表）的具体字段定义。
+ **设计依据**：概念设计产出的E-R图，以及所选用的数据模型（如关系模型）。
+ **典型特征**：**与具体数据库管理系统（DBMS）的功能有关，但与物理存储硬件无关**。例如，需要确定每个表的主键（用技术ID还是业务编号）、外键、字段数据类型（如VARCHAR还是INT），并进行**规范化处理**（通常达到3NF）以减少数据冗余。
+ **关键活动**：将实体转为表、将联系转为外键列、定义字段域名和数据类型、制定数据完整性规则（实体完整性、参照完整性、用户定义完整性）。

### 物理设计
+ **核心目标**：为逻辑设计好的表结构，在具体的物理存储介质上规划最合适的存取方式和存储结构，以达到最佳的性能。
+ **主要产出物**：**存储结构设计**、**索引设计**、分区策略、表空间分配方案。
+ **关注视角**：**底层硬件和性能视角**。重点考虑数据放在哪个磁盘、建什么类型的索引（B树/哈希）、表是否要按日期分区、日志和数据的存储分离等。
+ **设计依据**：逻辑设计的关系模式，以及具体的DBMS特性、服务器硬件配置、业务负载特征（读多写少或反之）。
+ **典型特征**：**高度依赖于具体的数据库软件和硬件环境**。同样的逻辑表，在MySQL和Oracle中的物理设计完全不同；同样，固态硬盘（SSD）和机械硬盘（HDD）上的索引策略也不同。
+ **关键活动**：确定表的存储引擎（如InnoDB）、建立合理索引（聚簇索引/二级索引）、设计数据分区策略、规划表空间和日志文件的物理存放路径、评估存储容量。

综合对比表

| 对比维度 | **概念设计** | **逻辑设计** | **物理设计** |
| :--- | :--- | :--- | :--- |
| **回答的问题** | 业务中有哪些事物？它们是什么关系？ | 这些关系用什么样的表结构来存？ | 这些表在硬盘上怎么存才能跑得快？ |
| **抽象层次** | 最高（业务抽象） | 中层（逻辑抽象） | 最低（物理抽象） |
| **依赖对象** | 只依赖业务需求 | 依赖DBMS的功能（如是否支持外键） | 依赖DBMS参数、硬件（CPU/内存/磁盘） |
| **主要工具/模型** | E-R图（实体关系图） | 关系模式（Table Schema）、规范化理论 | 索引、分区表、表空间、存储参数 |
| **设计者角色** | 系统分析员 + 业务专家 | 系统分析师 / 数据库架构师 | DBA（数据库管理员） |
| **变更影响** | 业务变更才改，影响最大 | 结构变更影响应用层代码 | 调整索引/分区不影响应用逻辑，只影响性能 |

## 规范化理论
### 教学内容
+ 函数依赖的概念与推导
+ 1NF → 2NF → 3NF 的判断与分解
+ 规范化原则：“尽可能高，但适可而止”
+ 反规范化的场景与权衡



### 范式分析
会议室预约系统范式分析

+ 所有表均已满足1NF（属性不可分）
+ 检查预约表：主键为`reservation_id`，所有非主属性完全依赖于主键 → 满足2NF
+ 检查预约表：无传递依赖（如`room_id`→`capacity`不存在，容量在会议室表中）→ 满足3NF
+ 员工表：`email`为候选键，无传递依赖 → 满足3NF

结论：当前设计满足3NF，无需进一步分解

## 物理设计
### 教学内容
+ 选择存储引擎（MySQL 8.0默认InnoDB）
+ 字段类型选择策略
+ 字符集选择：utf8mb4



### 物理设计决策
**物理设计决策**

| 表名 | 存储引擎 | 字符集 | 说明 |
| :--- | :--- | :--- | :--- |
| employee | InnoDB | utf8mb4 | 支持员工姓名中的特殊字符 |
| meeting_room | InnoDB | utf8mb4 |  |
| reservation | InnoDB | utf8mb4 |  |
| check_in | InnoDB | utf8mb4 |  |


**索引策略：**

+ `employee.email`：唯一索引（用于登录）
+ `reservation.employee_id`：普通索引（查询某员工的预约）
+ `reservation.room_id`：普通索引（查询某会议室的预约）
+ `reservation.start_time`、`reservation.end_time`：普通索引（时间范围查询）
+ `reservation.status`：普通索引（按状态筛选）
+ `reservation (room_id, start_time, end_time)`：复合索引（冲突检测优化）

### 建表
```sql
-- ============================================================
-- 1. 员工表 (employee)
-- ============================================================
CREATE TABLE employee (
    employee_id   VARCHAR(20)   PRIMARY KEY COMMENT '员工编号',
    name          VARCHAR(50)   NOT NULL COMMENT '姓名',
    department    VARCHAR(50)   NOT NULL COMMENT '部门',
    email         VARCHAR(100)  NOT NULL UNIQUE COMMENT '邮箱（登录账号）',
    phone         VARCHAR(20)   COMMENT '电话',
    role          ENUM('admin', 'employee') DEFAULT 'employee' COMMENT '角色',
    password_hash VARCHAR(255)  NOT NULL COMMENT '密码哈希',
    created_at    DATETIME      DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at    DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    is_active     BOOLEAN       DEFAULT TRUE COMMENT '是否启用'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='员工表';

-- ============================================================
-- 2. 会议室表 (meeting_room)
-- ============================================================
CREATE TABLE meeting_room (
    room_id       INT           PRIMARY KEY AUTO_INCREMENT COMMENT '会议室编号',
    room_name     VARCHAR(100)  NOT NULL COMMENT '会议室名称',
    floor         INT           NOT NULL COMMENT '所在楼层',
    capacity      INT           NOT NULL CHECK (capacity >= 1) COMMENT '最大容量',
    equipment     VARCHAR(200)  COMMENT '设备清单（投影仪/视频会议/白板等）',
    status        ENUM('available', 'maintenance', 'closed') DEFAULT 'available' COMMENT '状态',
    description   VARCHAR(255)  COMMENT '备注说明',
    created_at    DATETIME      DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at    DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='会议室表';

-- ============================================================
-- 3. 预约表 (reservation)
-- ============================================================
CREATE TABLE reservation (
    reservation_id   INT           PRIMARY KEY AUTO_INCREMENT COMMENT '预约编号',
    employee_id      VARCHAR(20)   NOT NULL COMMENT '预约人编号',
    room_id          INT           NOT NULL COMMENT '会议室编号',
    subject          VARCHAR(200)  NOT NULL COMMENT '会议主题',
    start_time       DATETIME      NOT NULL COMMENT '开始时间',
    end_time         DATETIME      NOT NULL COMMENT '结束时间',
    status           ENUM('pending', 'approved', 'rejected', 'cancelled', 'completed') 
                     DEFAULT 'pending' COMMENT '状态',
    created_at       DATETIME      DEFAULT CURRENT_TIMESTAMP COMMENT '申请时间',
    approved_at      DATETIME      NULL COMMENT '审批时间',
    rejected_reason  VARCHAR(255)  NULL COMMENT '拒绝原因',
    -- 外键约束
    CONSTRAINT fk_reservation_employee 
        FOREIGN KEY (employee_id) REFERENCES employee(employee_id),
    CONSTRAINT fk_reservation_room 
        FOREIGN KEY (room_id) REFERENCES meeting_room(room_id),
    -- 业务约束：开始时间必须早于结束时间
    CONSTRAINT chk_reservation_time CHECK (start_time < end_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='预约表';

-- ============================================================
-- 4. 签到表 (check_in)
-- ============================================================
CREATE TABLE check_in (
    check_in_id        INT           PRIMARY KEY AUTO_INCREMENT COMMENT '签到编号',
    reservation_id     INT           NOT NULL UNIQUE COMMENT '预约编号（一对一）',
    check_in_time      DATETIME      DEFAULT CURRENT_TIMESTAMP COMMENT '签到时间',
    actual_attendees   INT           COMMENT '实际参会人数',
    notes              VARCHAR(255)  COMMENT '备注',
    CONSTRAINT fk_checkin_reservation 
        FOREIGN KEY (reservation_id) REFERENCES reservation(reservation_id) 
        ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='签到表';
```



### 建索引
```sql
-- ============================================================
-- 5. 创建索引（性能优化）
-- ============================================================

-- 员工表索引
CREATE INDEX idx_employee_department ON employee(department);
CREATE INDEX idx_employee_name ON employee(name);

-- 预约表索引
CREATE INDEX idx_reservation_employee_id ON reservation(employee_id);
CREATE INDEX idx_reservation_room_id ON reservation(room_id);
CREATE INDEX idx_reservation_status ON reservation(status);
CREATE INDEX idx_reservation_start_time ON reservation(start_time);
CREATE INDEX idx_reservation_end_time ON reservation(end_time);

-- 复合索引：用于冲突检测（查询某会议室在某个时间段是否已被预约）
CREATE INDEX idx_reservation_room_time ON reservation(room_id, start_time, end_time);
```
